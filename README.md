# サーブレット & JSP 入門演習

## 演習1：`@WebServlet("/Yattemiru")`(servletTry.java)

### 🎯 目的
サーブレットの基本機能を習得し、**ユーザーのリクエストに応じてサーバー側のデータを動的に表示する**Webアプリケーションを構築します。

---

### 💻 アプリの概要
サーバー上に保存されたCSVデータを読み込み、ユーザーが指定した日付に基づいて情報を検索・表示します。

####  処理の流れ
| フェーズ | 役割 | 内容 |
| :--- | :--- | :--- |
| **準備** | データ管理 | サーバー側にCSV形式のデータファイルを配置 |
| **初期化 (`init`)** | データロード | アプリ起動時にCSVを読み込み、データクラスに格納 |
| **表示 (`doGet`)** | 入力画面 | 日付入力フォームと検索ボタンをブラウザに表示 |
| **実行 (`doPost`)** | 検索・出力 | 指定された日付をキーにデータを抽出、結果をWebページに反映 |

#### 🛠️ JSP研修後の演習としてJSP版にアップデート(3/1)
| 種別 | ソース |
| :--- | :--- | 
| 本体プログラム | servletJSP.java |
| **JSP** | servletJSP.jsp |

## 演習2：`@WebServlet("/population")`(PopulationServlet.java)

### 🎯 目的
1の応用で、**ユーザーのリクエストに応じて外部サーバーからデータを取得、加工して表示する** Webアプリケーションを構築します。<br>
フォームへのテキスト入力に対して、クロスサイトスクリプティング対策も実装しました。

### 💻 アプリの概要
フォームで入力された市町村名とサーバー上に保存されたCSVデータからコードを特定し、e-StatのAPIを使って該当市町村の人口を取得
Webページに表示表示します。

#### 🛠️ 処理の流れ
| フェーズ | 役割 | 内容 |
| :--- | :--- | :--- |
| **準備** | 市町村コード取得 | e-Statサイトからデータを取得(取得方法はAIに相談)<br>市町村名と都市コードを抜き出してCSV形式ファイルにしてサーバー側に配置 |
| **初期化 (`init`)** | データロード | アプリ起動時にCSVを読み込み、データクラスに格納 |
| **表示 (`doGet`)** | 入力画面 | 日付入力フォームと検索ボタンをブラウザに表示 |

## 演習3：`@WebServlet("/UserLogin")`(UserLogin.java)

### 🎯 目的
テキストのセッションスコープ講義のォームを使ったユーザー登録をするサンプルアプリが実際はなにも登録しないので、それに登録部分を実装し、さらにログイン画面も追加して、アプリとして形にしてみることにした。<br>

### 💻 アプリの概要
データの保存先としてサーバーにcsvファイルを置いて、登録でそのファイルにデータを追加し、ログインではそのファイルの情報から、アクセス可否を判断する<br>
セッションスコープの演習目的だが、CSVデータ操作には落とし穴（改行かカンマなど）があることと、その回避方法としてOpenCSVライブラリを利用することを同時に学ぶ。
テキストの範囲外なので、基本はAIと壁打ちしながら学習を進める。

#### 🛠️ 進捗状況
ログインやユーザー登録は単独で動作できること個まで完成したが、OpenCSVライブラリの利用や、ログインやユーザー登録に加え、ユーザー情報の表示や修正まで行えるアプリとしてリファクタリングを実施中
現在はDAOの買うメソッドの実装中
